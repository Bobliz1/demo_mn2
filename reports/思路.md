# 实验扩展思路

## 1. 增加函数的多样性和数量

在当前 `deceptive_trap_func`（欺骗性陷阱函数）基础上，扩充更多具有不同特性的测试函数，以增强实验的普适性和说服力。

**候选方向：**

- **多峰函数**：如 Rastrigin、Ackley，考验全局搜索能力
- **平坦区域 + 尖峰函数**：类似当前函数，但峰宽/峰高比例不同
- **非对称函数**：左右两侧梯度差异显著
- **高噪声函数**：信噪比低，考验 GP 的鲁棒性
- **多维扩展**：将当前 1D 函数推广到 2D/3D，验证高维场景下变换的有效性

**目标：** 覆盖"平坦陷阱"、"尖锐全局最优"、"多局部最优"等典型困难场景。

## 2. 扩充变换算子

当前变换库（`bo_transform.py`）仅包含 `LogWarper`、`StandardScaler`、`IdentityOperator`，需系统性扩充。

**候选算子：**

| 算子名称 | 描述 | 适用场景 |
|----------|------|----------|
| `PowerTransform` | $y' = y^\lambda$，Box-Cox / Yeo-Johnson | 处理右偏分布 |
| `RankTransform` | 将 y 值映射为百分位秩 | 消除异常值影响 |
| `SigmoidWarper` | $y' = 1/(1+e^{-y})$ | 压缩极端值范围 |
| `QuantileTransform` | 映射到均匀/正态分布 | 非参数化归一化 |
| `DifferenceTransform` | $y' = y - \bar{y}_{prev}$ | 强调局部改进量 |
| `MinMaxScaler` | $y' = (y - y_{min})/(y_{max} - y_{min})$ | 固定区间归一化 |

**流水线组合策略：**
- 当前：外层 GP 用二进制开关（Log on/off × Scale on/off）搜索 4 种组合
- 扩展后：将算子选择编码为连续/离散参数，支持更大的组合空间（如 $2^k$ 或顺序排列）

## 3. 实验输出对比：直接 BO vs. MN-BO

每次实验应同时输出两部分结果，形成**基线对比**：

### 3.1 基线：标准 BO（无变换）

- 使用相同初始点 `X_init`、`Y_init`
- 相同内层迭代次数 `N`
- 使用 `IdentityOperator`（即不做任何变换）直接优化
- 记录：最终误差 Gap，采样轨迹，收敛曲线

### 3.2 提出方法：MN-BO（带变换搜索）

- 外层 M 次迭代搜索最优变换配置
- 内层 N 次 BO 在变换空间中优化
- 记录：最终误差 Gap，最佳变换配置，收敛曲线

### 3.3 输出格式建议

```

[基线 BO]
  迭代次数：N = 15

  误差 Gap：x.xxxx

[MN-BO（本方法）]
  外层迭代 M = 5，内层迭代 N = 15
  最佳变换配置：Log=True, Scale=True
  搜索最大值：x.xxxx
  误差 Gap：x.xxxx

Gap 缩减比：xx.x%
========================
```

## 4. 外层 GP 优化器的改进

### 4.1 当前问题：连续 gate 导致搜索空间病态

当前外层搜索空间采用**软 gate 设计**（每个算子含 gate 参数 g∈[0,1]），存在以下系统性缺陷：

- **死维度**：当 gate g=0 时，其关联参数（如 LogWarper 的 α）对输出完全无影响，gap(p) 对该维度偏导为 0
- **空间病态**：gap(p) 在 g=0 附近几乎是分段常量，破坏了 GP 假设的平滑性，导致 GP 建模效率被系统性浪费
- **等价性弱**：在观测精度有限（内层 BO 仅 N_INNER=12 步 + 噪声）下，不同的 (g, α) 组合几乎不可区分

**具体冗余映射**：

| 算子 | gate 维度 | 被遮蔽维度 | 冗余条件 |
|------|---------|----------|--------|
| LogWarper | p[0] | p[1] (α) | p[0]=0 时 |
| StandardScaler | p[2] | p[3] (shift), p[4] (s) | p[2]=0 时 |
| PowerTransform | p[5] | p[6] (p) | p[5]=0 时 |
| SigmoidWarper | p[7] | p[8] (k), p[9] (c) | p[7]=0 时 |
| MinMaxScaler | p[10] | — | p[10]=0 时 |

### 4.2 改进方向：离散选择 + 连续调参（推荐）

**核心思路**：将"是否激活算子"从连续 gate 改为**离散选择**，激活后的参数用连续值调优。

**实现方案**：
- 外层参数空间改为：选择向量（哪个算子激活）+ 激活后各算子的连续参数
- 外层优化器从 sklearn GP 替换为支持**条件空间（Conditional Space）**的优化器
- 推荐工具：**SMAC（Sequential Model-based Algorithm Configuration）** 或 **ConfigSpace + HyPER**

**优点**：
- 消除死维度，搜索空间从 12D 有效降至 ~6D（仅激活的算子贡献参数）
- gap(p) 函数在条件子空间内连续可微，GP 建模效率大幅提升
- 离散选择对应算法层面的直观理解："用 LogWarper 还是 PowerTransform"

### 4.3 备选方案

| 方案 | 做法 | 优点 | 缺点 |
|------|------|------|------|
| **A. ARD 核函数** | GP 用各向异性 RBF，每个维度独立 length-scale | 实现简单，sklearn 原生支持 | 效率提升有限，死维度问题仍在 |
| **B. 接受冗余** | 保持当前设计，增加外层采样量（如 36→100） | 无需改代码 | 12D 稀疏采样仍效率低下 |
| **C. 边界约束替代 gate** | 去掉 gate，用 α=0.1 表示恒等 | 最简洁 | 仅适合 LogWarper，通用性差 |

**当前建议**：方案 B（接受冗余）适合论文验证阶段；方案 C（离散选择+SMAC）适合后续工程化改进。
