# M×N 双层贝叶斯优化（MN-BO）— 当前情况与架构

## 一、项目概述

**项目名称**：M×N 双层贝叶斯优化（MN-BO）
**项目路径**：`d:\2602-stu\paper\demo_mn2`
**核心目标**：验证 MN-BO 方法在欺骗性函数上的优化效果，通过外层 GP 搜索最优目标空间变换配置，内层标准 BO 执行优化。

---

## 二、技术架构

### 2.1 双层 BO 架构

```
┌─────────────────────────────────────────────────────────────┐
│                    外层 Meta-Level (M 次)                    │
│  ┌─────────────┐                                           │
│  │  外层 GP    │ ← 搜索变换参数 [log, scale] ∈ [0,1]²       │
│  │  (EI采集)   │ ← 目标：最小化 Gap (理论最大值 - 找到最大值) │
│  └──────┬──────┘                                           │
│         │ 建议下一组变换参数                                  │
│         ▼                                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              内层 Standard BO (N 次)                  │   │
│  │  Pipeline: LogWarper? → StandardScaler? → Identity   │   │
│  │  内层 GP (RBF核) + Expected Improvement 采集函数      │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

```
│   ├── periodic_trap.py       # 周期陷阱

│   ├── cliff.py               # 悬崖函数
│   ├── double_well.py         # 双势阱
│   ├── oscillating_decay.py   # 振荡衰减

│

```

### 2.3 核心类与组件

| 模块 | 类/函数 | 说明 |
|------|---------|------|
| `bo_transform.py` | `BaseOperator` | 变换算子基类 |
| | `LogWarper` | 对数变换 |
| | `StandardScaler` | 标准化变换 |
| | `IdentityOperator` | 恒等变换（不做任何变换） |
| | `TransformerPipeline` | 算子流水线（顺序执行 forward / backward） |
| | `WrappedTarget` | 包装后的目标函数（原始函数 + 变换管道） |
| `benchmark.py` | `expected_improvement()` | EI 采集函数 |
| | `propose_next_sample()` | 基于多重重启 L-BFGS-B 建议下一采样点 |
| | `ExperimentLogger` | 实验结果日志记录与可视化 |
| `functions/` | `BenchmarkFunc` | 所有基准函数的抽象基类 |

### 2.4 外层搜索空间

当前外层使用 4 个角点覆盖全部变换组合：

| 参数组合 | Log | Scale | 说明 |
|----------|-----|-------|------|
| `[0.0, 0.0]` | ❌ | ❌ | 恒等变换（基线） |
| `[1.0, 0.0]` | ✅ | ❌ | 仅对数变换 |
| `[0.0, 1.0]` | ❌ | ✅ | 仅标准化 |
| `[1.0, 1.0]` | ✅ | ✅ | 对数 + 标准化 |

## 五、当前实验结果

从 `summary_report.md` 可见，已完成部分实验：

| 函数 | 基线 Gap | MN-BO Gap | Gap 缩减比 | 最佳配置 |
|------|---------|----------|-----------|---------|
| `double_well_func` | 0.0340±0.0280 | 0.0061±0.0005 | **48.22%** | Log=False, Scale=False |
| `cliff_func` | 0.4924±0.4010 | 0.1945±0.4150 | **38.48%** | Log=True, Scale=False |
| `double_well_func` (重跑) | 0.0056±0.0001 | 0.0056±0.0000 | 0.45% | Log=False, Scale=True |

**观察**：MN-BO 在 `double_well_func` 和 `cliff_func` 上均展示了基线更优的优化效果。
