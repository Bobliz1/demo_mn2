# Antigravity 动态预算与早停机制 (Early Exit & Dynamic Budgeting) 设计方案

在 MN-BO 架构中，引入自适应预算分配的核心目的是实现**计算资源的按需流动**。通过实时监控搜索状态，使算法具备“止损”与“增加投资”的能力。

---

## 1. 三大核心场景逻辑

根据讨论，我们将固定步数 $N$ 升级为动态预算 $N_{adaptive}$，涵盖以下场景：

### A. 及时止损 (Early Exit - Wasteland)
*   **场景描述**：如果一直探索没有成果，且预期后面一段时间也没有成果。
*   **判断标准**：
    1.  **无成果**：连续 $K$ 步（Patience=3）未能在原始物理空间提升 $y_{max}$。
    2.  **无预期**：当前变换空间内的 $EI_{max}$ 持续低于全局阈值 $\eta$。
*   **动作**：立即跳出当前内层循环，进入下一轮 $M_{i+1}$（更换视角）。

### B. 机会主义追加 (Overtime - Potential Peak)
*   **场景描述**：如果当前暂时没成果，但指标显示正处于“爆发前夜”，允许突破 $N$ 的限制多跑一点。
*   **判断标准**：
    1.  **趋势向上**：采样点正在向高 $y$ 区域聚集，或者采集函数 $EI$ 的热度中心高度集中且数值依然维持在高位。
    2.  **潜力巨大**：$EI_{max}$ 远高于均值，说明存在尚未触达的肥沃区域。
*   **动作**：允许 $N$ 自动延展（例如最多延长 50% 的步数），直到满足退出条件。

### C. 见好就收 (Early Exit - Converged)
*   **场景描述**：已经有成果（即 $y_{max}$ 涨了），但后续提升非常微小，说明进入了平台期。
*   **判断标准**：
    1.  **二阶导数下降**：最近两次提升的幅度 $\Delta y$ 呈指数级衰减。
    2.  **收敛信号**：$\Delta y / (y_{max} - y_{min}) < \epsilon$（例如小于 0.1%）。
*   **动作**：停止当前挖掘，切换到下一个 $M$ 以寻找其他可能更高的峰。

---

## 2. 测量公平性 (Measurement Fairness)

由于算子变换（Log, Power, Scaling）会改变搜索景观的几何特征，所有判断必须遵循以下原则：

### A. 物理空间度量
*   **原则**：所有的改善判断（$\Delta y$）必须基于 **原始目标函数空间**。
*   **理由**：无论中间空间如何扭曲，我们最终关心的物理量（如成功率、最大值）是唯一的真理标准。

### B. $EI$ 归一化 (Relative EI)
*   **原则**：由于 GP 在不同变换下的方差尺度不同，判断“预期收益”时需使用归一化指标：
    $$EI_{norm} = \frac{EI_{max}}{y_{max\_pool} - y_{min\_pool}}$$
*   **理由**：这消除了量纲干扰，使得在 $M_1$（对数变换）和 $M_2$（幂变换）下的“潜力”具有可比性。

## 3. 技术实现框架 (main.py 伪代码)

```python
for step in range(N_MAX):
    # 1. 采集新点并观察物理 y 值
    x_new, y_new = inner_bo.suggest_and_observe()
    
    # 2. 归一化当前潜力指标
    ei_norm = calculate_max_ei() / global_y_range
    
    # 3. 动态逻辑决策
    if step >= MIN_STEPS:
        # 场景 A & C: 停滞或收敛
        if is_stagnant(history) or is_converged(history):
            if ei_norm < EXIT_THRESHOLD:
                break # 及时止损或见好就收
                
        # 场景 B: 追击潜力点
        if is_in_high_potential_area(x_new, ei_norm):
            extra_budget += 1 # 允许加班
```

## 5. 动态基线对齐：评估早停的“含金量”

为了确保早停后的结果依然具有统计公平性，我们引入“动态锚点”机制。

### A. 实时公平对比 (Step-to-Step Fairness)
*   **机制**：由于我们已经实现了“全量基线历史缓存”，当 MN-BO 在第 $j$ 步被触发“早停”或“追加”时，系统将动态切片基线数据。
*   **意义**：这让我们能回答核心问题：“虽然我提前结束了，但在结束的那一刻，我是否已经显著优于同期的基线？”

### B. 核心评价指标：效率增益 (Efficiency Gain)
在论文中，我们可以量化以下指标：
1.  **等效步数节省**：达到基线 90% 成功率时，MN-BO 节省了多少百分比的步数。
2.  **即时胜率曲线**：绘制在每一个步数 $j$ 上，MN-BO 与基线的胜率差。
