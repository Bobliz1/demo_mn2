# MN-BO 项目 AI 协作规则 & 惯例

在本项目（demo_mn2）的所有开发和维护过程中，AI 助手必须绝对遵守以下操作规范与项目惯例：

## 核心行为守则
1. **状态同步（每日日志）**：对项目做出的任何代码修改、超参数调整或环境变更等，均必须在完成改动的同时，记录到 `states.md` 文件中。以当天的日期（如 `## 2026-05-06`）为标题，按天记录项目中完成的事情。
2. **历史保留（回退规范）**：如果有一些之前的改动被决定回退、作废或重构，**绝对不能**在 `states.md` 或其他记忆文档中直接删除原有的记录。必须使用 Markdown 的删除线（例如：`~~被废弃的改动~~`）将原本的文本划掉。这样做是为了保留完整的迭代历史和“踩坑”记录。

## 运行环境与执行惯例
- **Python Path**: 必须使用 `./.venv/bin/python`。这是一个指向 `bo_env` Conda 环境的快捷软连接。
- **避免环境激活**: 绝对不要尝试运行 `conda activate`。请始终使用上述直接路径来调用 Python。

## 项目结构
- `src/main.py`: 核心实验入口程序。
- `src/bo_transform.py`: 处理 8D 空间变换的核心逻辑。
- `results/`: 所有实验报告（Markdown）和可视化图表的输出目录。
- `states.md`: 项目的长期记忆中枢，AI 接手任务前必须阅读。

## 常用测试命令
- **运行全量函数实验**:
  ```bash
  ./.venv/bin/python src/main.py --all
  ```
- **运行单个指定函数**:
  ```bash
  ./.venv/bin/python src/main.py <function_name>
  ```
