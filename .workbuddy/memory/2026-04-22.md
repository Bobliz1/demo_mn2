## 报告格式重构 + 数值溢出修复（15:08-16:30）

### 问题
旧汇总报告出现天文数字（gap_diff ~10^200+），原因是：
- `LogWarper.backward(y_prime)` 当 y_prime~100 时，exp(100)≈2.7×10^43，远超所有测试函数的理论最大值
- `mnbo_gap = gmax - 2.7e43` 变负
- `gap_diff = base_gap - mnbo_gap` 变成天文正数
- 缩减比计算溢出 inf

### 修复（benchmark.py）
1. 新增 `_fmt(val)` 函数：val<0 → "N/A"（识别溢出），|val|≥1e4 → 科学计数法
2. 新增 `_fmt_gap_diff(val, base_gap)`：gap_diff<0 或 gap_diff>100×base_gap → "N/A"
3. `append_summary_row`：去除方差/最佳配置列，标题含日期时间，溢出显示 N/A
4. `log_experiment`（单函数报告）：去除方差列，标题含运行时间

### 修复（main.py: run_inner_bo）
- 加宽松安全 clamp：`y_max_safe = max(y_init.max()*100, 1e6)`，截断极端 exp 溢出
- 溢出值在报告层用 N/A 标记，而非在前端隐藏

### 当前报告格式
- 单函数报告：标题含日期时间，5行核心结果表（理论最大/基线Gap/MN-BO Gap/缩减差值/缩减比），无方差
- 汇总报告：日期时间表头，无方差/最佳配置列，溢出→N/A

### 已知数值溢出问题（待长期修复）
`double_well_func` 等函数 MN-BO Gap 显示 N/A 的原因：
- `LogWarper.backward(y_prime)` 在 y_prime~30+ 时 exp(y_prime) 达到 10^13，远超所有测试函数理论最大
- 外层 GP 学会「往极端 y_prime 方向搜索」→ MN-BO gap 变负
- 报告正确用 N/A 标记为无效结果
- **长期修复方向**：在 `WrappedTarget` 或 `propose_next_sample` 里 clip y_prime 到安全范围（如 [-50, 50]），或换用有界变换算子
- 正常结果：科学计数法（3位有效数字）；异常结果：N/A
